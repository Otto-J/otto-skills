# PDF 发票信息提取

使用 Node.js 本地处理 PDF 发票，提取金额和项目信息并重命名文件。

## 功能特点

- ✅ **纯本地处理**：无需云端 API，保护隐私
- ✅ **自动提取**：从 PDF 中提取发票金额、项目名称
- ✅ **智能识别**：自动识别价税合计（最大的金额）
- ✅ **防冲突**：文件名包含时间戳，避免重复覆盖
- ✅ **批量处理**：一次处理整个目录的所有 PDF

## 使用场景

- 报销发票整理
- 发票批量归档
- 电子发票分类

## 工作流程

### 1. 确认处理目录

询问用户要处理的 PDF 目录路径。

**默认路径示例**：
- `/path/to/invoices/2025/month-01`

### 2. 执行处理脚本

运行 `scripts/rename-invoices.mjs` 脚本：

```bash
cd ~/.claude/skills/pdf-extract/scripts
node rename-invoices.mjs
```

脚本会自动：
1. 扫描目录下所有 PDF 文件
2. 提取文本内容
3. 识别所有 ¥ 开头的金额
4. 取最大的金额作为价税合计
5. 识别项目名称（通常在 `*xxx*xxx` 格式中）
6. 重命名为 `金额-项目-时间戳.pdf`

### 3. 检查结果

脚本会显示：
- 总文件数
- 每个文件的提取结果（金额、项目、所有金额列表）
- 重命名进度
- 最终统计

## 输出格式

### 文件命名规则

```
{金额}-{项目名称}-{时间戳}.pdf
```

**示例**：
- `101.70-餐饮服务-1737361234567.pdf`
- `131.12-服装鞋帽-1737361234568.pdf`
- `2360.00-住宿服务-1737361234569.pdf`

### 时间戳说明

- 使用毫秒级时间戳（13位数字）
- 确保文件名唯一性
- 防止重复文件相互覆盖

## 技术实现

### 依赖库

- `pdfjs-dist`：Mozilla 官方 PDF 解析库（ES Module）
- Node.js 内置模块：`fs`, `path`

### 核心逻辑

#### 1. 金额提取

```javascript
// 提取所有 ¥ 开头的数字
const amountRegex = /¥\s*([\d,]+\.?\d*)/g;
// 过滤条件：
// - 金额 > 0
// - 金额 < 1,000,000（排除发票号码）
// - 数字字符串长度 < 15
// 去掉小数点后数字不超过 12 位
// 取最大值作为价税合计
```

#### 2. 项目名称提取

```javascript
// 匹配模式：*行业*商品名
const patterns = [
  /\*[^*]+\*\s*([^*\s]{2,})/,
  /\*[^*]+\*([^*\s]+)/,
];
// 项目名称长度：2-30 字符
```

#### 3. 防冲突策略

```javascript
// 第一步：重命名为临时文件名
temp_0_原文件名.pdf
temp_1_原文件名.pdf
// ...

// 第二步：提取信息

// 第三步：重命名为最终格式（带时间戳）
金额-项目-时间戳.pdf
```

## 限制和注意事项

### ✅ 能做到

- 提取文本型 PDF 的文字内容（100% 成功率）
- 识别发票金额和项目（约 93% 准确率）
- 处理电子发票、数码发票
- 批量处理大量文件

### ⚠️ 无法做到

- **扫描件/图片 PDF**：需要 OCR 技术
  - 本地方案：Tesseract.js（准确率 60-70%）
  - 云端方案：百度/腾讯/阿里 OCR（准确率 95%）
- **复杂布局发票**：可能影响字段识别
- **手写文字**：需要 OCR 支持

### ❌ 已知问题

1. **部分发票文本缺失**
   - 原因：PDF 可能是图片或特殊格式
   - 表现：提取的文本很少，无法识别金额和项目
   - 处理：标记为"解析失败"，保留原文件名

2. **发票号码误识别**
   - 原因：某些发票的发票号码紧跟 ¥ 符号
   - 解决：添加长度过滤（数字字符串 < 15 位）

3. **文件已存在冲突**
   - 原因：多个发票有相同金额和项目
   - 解决：使用时间戳确保唯一性

## 故障排查

### 问题：npm install 失败

```bash
# 清理缓存重试
rm -rf node_modules package-lock.json
npm install pdfjs-dist
```

### 问题：脚本运行报错

检查 Node.js 版本：
```bash
node --version  # 需要 >= 14.0.0
```

### 问题：识别准确率低

- 确认 PDF 是文本型而非扫描件
- 检查 PDF 是否加密
- 尝试用 PDF 编辑器重新保存
