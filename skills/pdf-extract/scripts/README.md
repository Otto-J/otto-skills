# PDF 发票信息提取工具

从 PDF 发票中自动提取金额和项目信息，并重命名为 `金额-项目-时间戳.pdf` 格式。

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 设置发票目录

**方式一：修改脚本默认目录**

编辑 `rename-invoices.mjs` 第 20 行：

```javascript
const PDF_DIR = '/你的/发票/目录/路径';
```

**方式二：使用环境变量**

```bash
export PDF_DIR=/你的/发票/目录/路径
node rename-invoices.mjs
```

### 3. 运行

```bash
node rename-invoices.mjs
```

## 输出示例

处理前：
```
invoice-sample-001.pdf
invoice-sample-002.pdf
invoice-sample-003.pdf
```

处理后：
```
101.70-餐饮服务-1737361234567.pdf
131.12-服装鞋帽-1737361234568.pdf
2360.00-住宿服务-1737361234569.pdf
```

## 功能特点

- ✅ 纯本地处理，无需云端 API
- ✅ 自动识别最大金额（价税合计）
- ✅ 提取项目名称
- ✅ 时间戳防冲突
- ✅ 批量处理

## 技术细节

### 金额识别逻辑

1. 提取所有 `¥` 开头的数字
2. 过滤无效金额：
   - 金额 < 100万
   - 数字字符串长度 < 15
   - 去掉小数点后不超过 12 位
3. 取最大值作为价税合计

### 项目名称识别

匹配模式：`*行业*商品名`

示例：
- `*餐饮服务*餐饮服务` → `餐饮服务`
- `*服装*服装鞋帽` → `服装鞋帽`

### 时间戳

使用毫秒级时间戳（13位）+ 索引，确保唯一性。

## 依赖

- Node.js >= 14.0.0
- pdfjs-dist (Mozilla 官方 PDF 解析库)

## 故障排查

### 识别准确率低

- 确认 PDF 是文本型而非扫描件
- 检查 PDF 是否加密

### 部分文件失败

正常现象，可能原因：
- PDF 是扫描件/图片（需要 OCR）
- PDF 文本内容缺失
- 特殊格式发票

失败的文件会保留原文件名。
